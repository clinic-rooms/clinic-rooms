# מסלול א: התקנה בדפדפן (מומלץ, כ-10 דקות)

## מה צריך

| חשבון | בשביל מה | עלות |
|---|---|---|
| **GitHub** | שומר את קוד המערכת של המרפאה ומאפשר עדכונים | חינם |
| **Vercel** | מארח את המערכת באינטרנט | חינם |

## שלב 1: פתיחת חשבונות (חד-פעמי, ~3 דקות)

1. גלשו אל <https://github.com> → **Sign up** → הירשמו עם כתובת מייל.
2. גלשו אל <https://vercel.com/signup> → בחרו **Continue with GitHub**
   (חשוב! כך שני החשבונות מקושרים).

## שלב 2: לחיצה על כפתור ההתקנה

> ⚠️ **מתקינים עבור מרפאה, ויש לכם כבר חשבון Vercel/GitHub אישי?**
> העותק של המערכת נוצר בחשבון ה-GitHub שמקושר לחשבון ה-Vercel שאתם מחוברים
> אליו ברגע הלחיצה. כדי שהמערכת תשב בבעלות המרפאה (ולא בחשבון הפרטי שלכם) —
> התנתקו קודם מהחשבונות האישיים, והתחברו עם חשבונות המרפאה משלב 1.

1. בעמוד הראשי של המאגר (README) לחצו על כפתור **Deploy with Vercel**.
2. ייפתח טופס של Vercel:
   - **Repository Name:** השאירו `clinic-rooms` (או שנו) — זה העותק הפרטי של המרפאה.
   - **Neon:** אשרו את יצירת מסד הנתונים (מוצע אוטומטית; בחרו את התוכנית החינמית).
     ⚠️ **חשוב — Region:** אם מוצעת בחירת אזור למסד, בחרו
     **Frankfurt (eu-central-1)**. האפליקציה עצמה מוגדרת מראש לרוץ בפרנקפורט —
     מסד באותו אזור = מערכת מהירה; מסד בארה"ב = כל פעולה איטית מורגשת.
   - **BETTER_AUTH_SECRET** — ראו את המסגרת הבאה 👇
3. לחצו **Deploy** והמתינו 2–3 דקות.

> ### 🔑 השדה של "מפתח האבטחה" — מה עושים שם?
>
> זה השדה הידני **היחיד** בכל ההתקנה, והוא פשוט יותר ממה שהוא נראה:
>
> **מה זה?** סיסמה פנימית של המערכת (לא שלכם!) שאיתה היא חותמת את
> ההתחברויות ומצפינה מידע רגיש. אף אחד לא יקליד אותה שוב — היא רק צריכה
> להיות ארוכה, אקראית וסודית.
>
> **מה עושים?** מקלידים בשדה **ג'יבריש חופשי של 40 תווים או יותר** —
> אותיות באנגלית ומספרים בערבוביה. אין צורך לזכור או להמציא משהו חכם;
> אפשר ממש "לשבת על המקלדת". זה צריך להיראות בסגנון:
>
> ```
> t7Kp2mXw9qRj4nBv6cLd8sFh3gYz5aEi1uJo0eNs7hQf2mTp9xW
> ```
>
> ⚠️ **אל תעתיקו את הדוגמה** — הקלידו משהו אקראי משלכם.
>
> **ולסיום:** העתיקו את מה שהקלדתם ושמרו במקום בטוח (פתק בטלפון, מנהל
> סיסמאות). ביום-יום לא תצטרכו אותו — הוא נחוץ רק אם תקימו את הפרויקט
> מחדש אי-פעם.

## שלב 3: הגדרה ראשונית — בתוך המערכת

1. בסיום הפריסה לחצו על הקישור/התמונה — נפתחת המערכת בכתובת הקבועה שלה
   (בסגנון `https://clinic-rooms-xxxx.vercel.app`). **זו הכתובת שתמסרו לצוות.**
2. מסך "ברוכים הבאים" — יוצרים את **חשבון הניהול הראשון** (שם מלא, שם משתמש
   באנגלית, סיסמה). המסך הזה זמין רק כשהמערכת ריקה — אין סכנה שזר ישתמש בו אחר כך.
3. מתחברים, ונפתח **אשף הגדרה בעברית**: אישור תנאי השימוש, שם המרפאה, ימי
   ושעות פעילות, חדרים ואנשי צוות. הכול ניתן לשינוי אחר-כך בהגדרות.

**זהו — המערכת מוכנה.** מסרו לכל איש צוות את הכתובת + שם המשתמש והסיסמה הזמנית שלו.

## שלב 4: הפעלת עדכונים אוטומטיים (חד-פעמי, שתי לחיצות)

תהליך היצירה של Vercel משמיט מטעמי אבטחה את קובץ העדכון האוטומטי — לכן
מפעילים אותו פעם אחת. **אשף ההגדרה מציע את זה בעצמו במסך הסיום** — לחצו שם
על הכפתור, וב-GitHub שנפתח לחצו **Commit changes** (הכפתור הירוק). זהו.

דילגתם? אפשר תמיד מאוחר יותר: ניהול → **הגדרות** → כרטיס **"עדכונים
אוטומטיים"** → אותן שתי לחיצות.

מרגע זה המערכת בודקת כל לילה אם יש גרסה חדשה ומתעדכנת לבד.

## תוספות (רשות, אפשר בכל שלב)

- **עוזר חכם (Claude):** ניהול → הגדרות → כרטיס "בינה מלאכותית" — הדביקו מפתח
  מ-<https://console.anthropic.com> (נשמר מוצפן). בלי Vercel, בלי טכנולוגיה.
  שימו לב: זה השירות היחיד **בתשלום לפי שימוש** — אך העלות מזערית: כ-0.2–0.5 ₪
  לשאלה לעוזר, ובחודש טיפוסי שקלים בודדים. מומלץ להגדיר תקרת הוצאה ב-Billing.
  פירוט מלא במדריך המנהל.
- **גיבוי יומי ל-GitHub:** צרו מאגר פרטי ריק בשם `clinic-rooms-backups`, הפיקו
  [Fine-grained Token](https://github.com/settings/personal-access-tokens/new)
  (גישה רק למאגר הזה, הרשאת Contents: Read and write), ובדשבורד Vercel →
  Settings → Environment Variables הוסיפו: `GITHUB_BACKUP_TOKEN`,
  `GITHUB_BACKUP_REPO` (בפורמט `שם-משתמש/clinic-rooms-backups`) ו-`CRON_SECRET`
  (טקסט אקראי) → Redeploy. הגיבוי ירוץ כל לילה. (בנוסף, Neon שומר תמיד 24 שעות אחורה.)
- **התראות דחיפה:** הריצו במחשב `npx web-push generate-vapid-keys` והוסיפו
  ב-Vercel את `NEXT_PUBLIC_VAPID_PUBLIC_KEY`, `VAPID_PRIVATE_KEY` ו-`VAPID_SUBJECT`
  (בפורמט `mailto:המייל@שלכם`) → Redeploy.
- **דומיין משלכם:** Vercel → Settings → Domains → Add, ואז הוסיפו משתנה
  `BETTER_AUTH_URL` עם הכתובת החדשה → Redeploy.

## עדכוני גרסה — אוטומטיים

אחרי ההפעלה החד-פעמית (שלב 4 למעלה; במסלול ב זה מגיע מובנה), המערכת
**מתעדכנת לבד**: כל לילה (03:30, כשהמרפאה סגורה) נבדק אם יש גרסה
חדשה במאגר המקור, ואם כן — היא נפרסת אוטומטית. אחרי עדכון, כל משתמש רואה
בכניסה הבאה מסך "מה חדש" עם השינויים.

- **עדכון מיידי ידני:** GitHub של המרפאה → Actions → עדכון מערכת → Run workflow.
- **ביטול העדכון האוטומטי** (למי שמעדיף ידני): בקובץ
  `.github/workflows/update.yml` הוסיפו `#` בתחילת שתי שורות ה-`schedule`.
- **משהו השתבש אחרי עדכון?** דשבורד Vercel → Deployments → ⋯ על הגרסה
  הקודמת → **Rollback** — חזרה מיידית. (הנתונים לעולם אינם חלק מהעדכון.)

# מסלול ב: התקנה מקומית מהמחשב (חלופי)

מתאים למי שמעדיף אשף מודרך מהמחשב, או רוצה לבדוק מקומית לפני העלאה.

1. התקינו [Node.js](https://nodejs.org) (גרסת LTS).
2. הורידו את הקוד (Code → Download ZIP) וחלצו לתיקייה נוחה.
3. **Windows:** לחיצה כפולה על `install.bat` · **Mac:** `bash install.sh`.
4. נפתחים שני חלונות: **מדריך מלא בעברית בדפדפן** (`setup/guide.html`) וחלון
   פקודות שמציג שורות קצרות באנגלית — מספרי ה-STEP תואמים לסעיפי המדריך.
   האשף מלווה אתכם דרך: מסד Neon, מפתחות, חשבון ניהול, בדיקה מקומית,
   והעלאה ל-GitHub + Vercel.

האשף שומר התקדמות — אפשר לעצור ולהריץ שוב מאותה נקודה.

# פתרון תקלות

| בעיה | פתרון |
|---|---|
| המערכת מרגישה איטית | כנראה המסד נוצר בארה"ב במקום בפרנקפורט. בדשבורד Vercel → Storage בדקו את ה-Region של Neon; אם אינו eu-central-1 — צרו מסד חדש בפרנקפורט (Storage → Create Database → Neon), נתקו את הישן, ועשו Redeploy. אם כבר הוזנו נתונים — הורידו קודם גיבוי מהמסך ושחזרו אחרי |
| הכפתור נפתח בלי הצעת מסד נתונים | ודאו שאתם מחוברים ל-Vercel; אפשר גם להוסיף אחרי הפריסה: Storage → Create Database → Neon |
| הפריסה נכשלה (Build Failed) | פתחו את הלוג ב-Vercel; אם השגיאה מזכירה migrate/DATABASE_URL — ודאו שמסד Neon מחובר בפרויקט (Storage) ועשו Redeploy |
| נכנסתי לכתובת ורואה מסך התחברות במקום "ברוכים הבאים" | המערכת כבר הוגדרה — התחברו עם החשבון שנוצר. שכחתם? ראו שורה הבאה |
| שכחתי את סיסמת הניהול | מנהל אחר מאפס ממסך ניהול המשתמשים; אם אין מנהל נוסף — פנו למי שסיפק את המערכת |
| העוזר החכם לא עובד | ודאו שהמתג דלוק ושהוזן מפתח בכרטיס הבינה המלאכותית (ניהול → הגדרות) |
| לחיצה על install.bat לא עושה כלום (מסלול ב) | ודאו שחילצתם את ה-ZIP; אם מופיע "Windows protected your PC" — More info → Run anyway |
| "Database connection failed" (מסלול ב) | העתיקו שוב את כתובת ה-Pooled connection מ-Neon, את כל השורה |
| המדריך לא נפתח בדפדפן (מסלול ב) | פתחו ידנית את `setup\guide.html` מהתיקייה |
