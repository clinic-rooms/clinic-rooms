# מדריך למנהל/ת המרפאה

מדריך זה מסביר את כל מסכי הניהול. לכניסה למסכים אלה נדרש חשבון עם הרשאת ניהול.

---

## כניסה ראשונה — אשף ההגדרה

בכניסה הראשונה של מנהל למערכת חדשה נפתח אשף קצר:

1. **שם המרפאה** — יופיע בכותרות, במסך ההתחברות ובאייקון בטלפון.
2. **ימי פעילות** — אילו ימים מופיעים בלוחות (שבת לא קיימת במערכת).
3. **שעות פעילות** — טווח היום בלוח, במשבצות של חצי שעה (עד 15.5 שעות).
4. **חדרים** — שם + מאפיינים (חלון, כיור, גדול, קבוצות). כל חדר נפתח
   אוטומטית בכל ימי ושעות הפעילות; דיוק חלונות זמינות — במסך ניהול החדרים.
5. **אנשי צוות** — שם, שם משתמש וסיסמה זמנית לכל אחד/ת. רשמו את הפרטים
   ומסרו אישית — בכניסה הראשונה כל אחד/ת מחליף/ה לסיסמה אישית.
6. **החשבון שלכם** — שינוי השם ושם המשתמש שנקבעו בהתקנה, והחלפת סיסמה.

הכול ניתן לשינוי אחר-כך במסכי הניהול. בסיום — "כניסה למערכת".

---

## מסך הניהול הראשי — לוח החדרים

תצוגת היום: עמודה לכל חדר, שורה לכל חצי שעה. פס "חום" עליון מציג עומס תפוסה.

**לחיצה על תא פנוי:**
- **שיבוץ חד-פעמי** — הזמנה למטפל/ת ליום זה בלבד.
- **שיבוץ קבוע** — חוזר כל שבוע באותו יום מאותו תאריך.
- **טקסט חופשי** — כותרת לחדר בלי חשבון משתמש (מטפל אורח, שם קבוצה).
  החדר נחשב תפוס בשעות אלה.

**לחיצה על תא תפוס:**
- ביטול/סיום/מחיקת השיבוץ (סיום = מתאריך מסוים, ההיסטוריה נשמרת).
- **פינוי שעות** — חד-פעמי (מופיע כמקווקו) או קבוע (החלון חוזר להיות פנוי).
- **עריכת שעות** השיבוץ הקבוע.
- **שינוי מתוזמן** — "החל מתאריך X עוברים לחדר/יום/שעות אחרים". עד אז הכול
  כרגיל, ומהתאריך השינוי חל מעצמו.
- **שיבוץ כפול** — שני אנשי צוות באותו חדר (מוצג מפוצל; זמין רק לניהול).

חיפוש מעל הלוח: "איפה דנה עכשיו?" או "מי בחדר 3?".
עין ("הצגת חופשות ארוכות") מציגה שיבוצים של עובדים מושבתים כרקע.

## ניהול חדרים

- הוספה/עריכה: שם, חלון, כיור, גדול, חדר קבוצות, חדר חיצוני, הערות.
- **חלונות זמינות** — מתי החדר קיים בלוח. חדר בלי חלונות לא יופיע כלל.
  אפשר חלונות שונים לימים שונים (למשל חדר שקיים רק בבוקרי רביעי).
- השבתה (Power) — החדר נעלם מהלוח, הנתונים נשמרים.

## ניהול צוות

- **הוספה** — שם, שם משתמש, סיסמה זמנית, תפקיד (ניהול/צוות) ודרגה
  (צוות קבוע / מתמחה / סטודנט — פנימי, לא מוצג לצוות ומשמש רק כשיקול עדין בשיבוץ).
- **איפוס סיסמה** — קובע סיסמה זמנית חדשה; המשתמש יתבקש להחליפה בכניסה.
- **השבתה** — לחופשה ארוכה (לידה וכו'): החשבון ננעל, השיבוצים הקבועים מוצגים
  כ"רוחות רפאים" ברקע והחדר פנוי לאחרים. הפעלה מחדש מחזירה הכול.
- **מחיקה** — מוחקת גם את ההיסטוריה. עדיף השבתה.

## הגדרות

- **שם, ימי פעילות ושעות פעילות** — שינוי שעות מצומצם יותר מציג שיבוצים חורגים
  כחתוכים; שום נתון לא נמחק, והרחבה מחזירה אותם. המערכת מבקשת אישור לפני צמצום.
- **קישור צפייה** — קישור ציבורי ללוח במצב קריאה בלבד (למסך במרפאה למשל).
  ביטול הקישור חוסם מיד את הגישה.
- **מנהלים** — מינוי/הסרה של מנהלי מערכת נוספים.
- **בינה מלאכותית** — מתג הפעלה/כיבוי לכל פיצ'רי Claude.
- **ימי חג וסגירה** — חגי ישראל מזוהים אוטומטית (חג = סגור, ערב חג = עד 13:00).
  אפשר לשנות כל תאריך (סגור/מקוצר/פתוח כרגיל) או להוסיף סגירה ידנית (יום גשר).
- **גיבוי** — הורדת גיבוי מלא כקובץ, ושחזור מקובץ (זהירות: דורס את כל הנתונים).

## גיבויים — חשוב להכיר

לרשותכם שלוש שכבות גיבוי:

1. **גיבוי ידני מהמסך** — ניהול → הגדרות → גיבוי → "הורדת גיבוי". קובץ JSON
   אחד עם כל הנתונים. מומלץ להוריד לפני שינויים גדולים.
2. **גיבוי יומי אוטומטי ל-GitHub** (אם הוגדר בהתקנה) — כל לילה נשמר גיבוי
   מלא במאגר הפרטי `clinic-rooms-backups` בחשבון ה-GitHub של המרפאה.
   **איך ניגשים אליו:** github.com → התחברו → המאגר `clinic-rooms-backups` →
   תיקיית `backups` → קובץ לפי תאריך (`2026-07-21.json`) → כפתור Download.
   את הקובץ משחזרים מהמסך: ניהול → הגדרות → גיבוי → "שחזור מקובץ".
3. **שחזור נקודת-זמן ב-Neon** — בדשבורד של Neon (חשבון המרפאה) אפשר לשחזר
   את המסד לכל רגע ב-24 השעות האחרונות (Restore) — למקרה של טעות שקרתה עכשיו.

⚠️ המערכת מסופקת ללא אחריות וללא התחייבות לתחזוקה — **הגיבויים באחריותכם**.
מומלץ לוודא אחת לתקופה שהגיבוי היומי אכן נכתב (פשוט להיכנס למאגר ולראות
שיש קובץ מהיום).

## בקשות והחלפות

- **בקשות לשעה קבועה** — כשמטפל/ת מבקש/ת שיבוץ קבוע, הבקשה ממתינה לאישורכם
  (מסך ניהול המשתמשים/התראות). אישור יוצר את השיבוץ אוטומטית.
- **החלפות** — הצוות מסדר החלפות בעצמו; אתם רואים הכול בלוח ויכולים לבטל.

## סטטיסטיקות

תפוסה שבועית לכל חדר, אחוזי תפוסה בשעות השיא, והשעה העמוסה ביותר —
עוזר להחליט אם צריך עוד חדר או לפזר שעות.

## עדכוני גרסה

המערכת **מתעדכנת אוטומטית** כל לילה (03:30, כשהמרפאה סגורה) מגרסאות
שמתפרסמות במאגר המקור. אחרי עדכון, כל משתמש רואה פעם אחת מסך "מה חדש"
עם רשימת השינויים. הנתונים שלכם לעולם אינם חלק מהעדכון — רק הקוד מתעדכן.

- **עדכון מיידי בלי לחכות ללילה:** GitHub של המרפאה → Actions →
  "עדכון מערכת" → Run workflow.
- **עדכון גרם לבעיה?** דשבורד Vercel → Deployments → ⋯ על הגרסה הקודמת →
  **Rollback** — חזרה מיידית לגרסה הקודמת.
- **מעדיפים לעדכן רק ידנית?** בקובץ `.github/workflows/update.yml` הוסיפו
  `#` בתחילת שתי שורות ה-`schedule`.

## העוזר החכם (Claude)

צ'אט ניהולי שמכיר את הלוח בזמן אמת: "איפה אפשר לשבץ מתמחה חדשה בימי א'+ג'
בבוקר?", "מה צריך לקרות כדי לפנות את חדר 2 ביום חמישי?".

- העוזר **בודק נתונים אמיתיים** ומציג תוכניות מדורגות.
- כששינוי מוצע — הוא מופיע ככרטיס עם כפתור אישור. **שום שינוי לא מבוצע בלי
  לחיצת האישור שלכם.**
- הפעלה: בהגדרות → כרטיס "בינה מלאכותית" — מדליקים את המתג ומדביקים מפתח
  API של Anthropic (נוצר ב-console.anthropic.com, נשמר מוצפן במערכת).
  אפשר להסיר או להחליף את המפתח בכל רגע מאותו מקום.

### כמה זה עולה?

השימוש במפתח Anthropic הוא **בתשלום לפי שימוש בפועל** — אבל בהיקפים של
מרפאה מדובר בסכומים מזעריים (הערכות, נכון ל-2026):

| שימוש | עלות משוערת |
|---|---|
| שאלה לעוזר החכם (כולל בדיקות הלוח שהוא מבצע) | ‎~0.2–0.5 ₪ |
| הבנת היעדרות בשפה חופשית | פחות מאגורה |
| **מרפאה טיפוסית בחודש** | **שקלים בודדים, לרוב מתחת ל-20 ₪** |

טיפים: בקונסולת Anthropic (Billing) רואים את הצריכה בפועל, ואפשר להגדיר
**תקרת הוצאה חודשית** — מומלץ להגדיר, ואז אין הפתעות. כשלא משתמשים — לא
משלמים כלום, וכל שאר המערכת לא תלויה בזה בכלל.
