מדריך התקנה — מערכת ניהול חדרי טיפול

השאירו את הדף הזה פתוח לצד החלון השחור — הוא מלווה אתכם שלב-שלב.

💡 איך זה עובד: החלון השחור (שורת הפקודה) מציג באנגלית פשוטה את מספר השלב, למשל STEP 3 — וכאן בדף מוצא לו הסבר מלא בעברית עם אותו מספר. בחלון השחור רק מדביקים ערכים ומקישים Enter.
קפיצה לשלב: תנאי שימוש 3 מסד נתונים 5 בינה מלאכותית 6 התראות 8 חשבון ניהול 9 בדיקה מקומית 10 GitHub 11 Vercel 12 גיבוי יומי אחרי ההתקנה פתרון תקלות

1לפני שמתחילים — תנאי שימוש 💙

המערכת הזו נבנתה בהתנדבות, כשירות לציבור — ללא כל מטרה מסחרית.

היא לא נכתבה על ידי מתכנתים מקצועיים, אלא בעזרת כלי בינה מלאכותית, תוך שימת דגש רב על אבטחת מידע ופרטיות.

עם זאת, המערכת מסופקת כמות-שהיא (As-Is), ללא אחריות מכל סוג — לא לתקלות, לא לאובדן מידע, ולא לכל נזק או סוגיה שתעלה מהשימוש בה.

האחריות המלאה על ההתקנה, הנתונים והשימוש — על המתקין והמפעיל בלבד. מומלץ מאוד להפעיל את הגיבוי היומי המובנה (שלב 12).

✍️ אם קראתם ואתם מסכימים — הקלידו בחלון השחור agree והקישו Enter.

2התקנת רכיבי תוכנה אוטומטי

המערכת מורידה את הרכיבים שהיא צריכה. אין מה לעשות — רק לחכות כמה דקות. ייתכן שהמסך יתמלא בשורות טקסט רצות — זה תקין.

3מסד נתונים — Neon חינם

כל הנתונים (חדרים, שיבוצים, משתמשים) נשמרים במסד נתונים בענן של Neon.

  1. גלשו אל neon.tech ולחצו Sign Up → התחברו עם חשבון Google (הכי פשוט).
  2. לחצו New Project ומלאו:
    • Project name: clinic-rooms (או כל שם)
    • Region: AWS Europe (Frankfurt) — הקרוב לישראל
  3. אחרי היצירה מופיע מסך Connect to your database. ודאו שמסומן Pooled connection.
  4. לחצו על כפתור ההעתקה ליד כתובת החיבור — שורה ארוכה שמתחילה ב-postgresql://
  5. עברו לחלון השחור, הדביקו (קליק ימני על החלון = הדבקה) והקישו Enter.
💾 שמרו את פרטי ההתחברות לאתר Neon — דרכו אפשר גם לשחזר נתונים עד 24 שעות אחורה במקרה של טעות.

4מפתחות אבטחה אוטומטי

נוצרים לבד ונשמרים בקובץ מקומי. אין מה לעשות 🙂

5בינה מלאכותית — Claude רשות

המערכת כוללת עוזר חכם לניהול (צ'אט שמציע שיבוצים) והבנת היעדרויות בשפה חופשית. זה דורש מפתח של Anthropic (תשלום קטן לפי שימוש). המערכת עובדת מצוין גם בלי זה — אפשר לדלג ולהוסיף בעתיד.

בחלון השחור: y = כן, להגדיר · n = דילוג.

אם בחרתם כן:

  1. גלשו אל console.anthropic.com והירשמו.
  2. ב-Billing הוסיפו אמצעי תשלום.
  3. ב-API Keys לחצו Create Key, העתיקו את המפתח (מתחיל ב-sk-ant-) והדביקו בחלון השחור.

6התראות לטלפון רשות

מאפשר לצוות לקבל התראות (החלפות, שיבוצים) גם כשהאפליקציה סגורה. הכול נוצר אוטומטית — רק תתבקשו להקליד את כתובת המייל שלכם.

7הקמת טבלאות המסד אוטומטי

המערכת בונה את הטבלאות במסד הנתונים שפתחתם בשלב 3. רק לחכות רגע.

8חשבון הניהול שלכם

בחלון השחור תתבקשו לקבוע:

  • Username — שם משתמש להתחברות, באנגלית קטנה (למשל dana).
  • Password — סיסמה, לפחות 8 תווים. בזמן ההקלדה יוצגו כוכביות.
🔑 שמרו את הסיסמה! היא לא נשמרת בשום קובץ ואין דרך לשחזר אותה בלי מנהל נוסף.

את שם המרפאה והשם שלכם בעברית תגדירו בתוך האפליקציה עצמה, מיד אחרי ההתחברות הראשונה — שם העברית עובדת מושלם.

9בדיקה מקומית

  1. האשף מפעיל את המערכת על המחשב שלכם.
  2. פתחו בדפדפן: localhost:3000
  3. התחברו עם שם המשתמש והסיסמה שקבעתם בשלב 8.
  4. ייפתח אשף הגדרה קצר בעברית (שם המרפאה, ימים, שעות, חדרים, צוות) — אפשר לעבור אותו עכשיו או אחרי ההעלאה לאינטרנט.
  5. כשסיימתם — חזרו לחלון השחור והקישו Ctrl+C לעצירת השרת, ואשרו שהכול עבד (y).

10העלאת הקוד — GitHub חינם

GitHub שומר את קוד המערכת בענן ומאפשר עדכונים אוטומטיים.

  1. אם אין לכם חשבון — הירשמו ב-github.com (חינם).
  2. אם מותקן אצלכם GitHub CLI — האשף יעשה הכול לבד (אולי ייפתח דפדפן לאישור התחברות).
  3. אחרת, האשף יבקש ליצור מאגר ידנית:
    • ב-GitHub לחצו + (למעלה) → New repository
    • Repository name: clinic-rooms · בחרו Private
    • אל תסמנו שום אפשרות Initialize — מאגר ריק לגמרי → Create repository
    • בחלון השחור הקלידו את שם המשתמש שלכם ב-GitHub. בדחיפה ייתכן שייפתח חלון התחברות של GitHub — אשרו בדפדפן.

11העלאה לאינטרנט — Vercel חינם

  1. אם אין לכם חשבון: גלשו אל vercel.com/signup ובחרו Continue with GitHub (חשוב! כך החשבונות מקושרים).
  2. האשף מתחבר (ייפתח דפדפן לאישור), יוצר פרויקט, מעלה את ההגדרות ופורס.
  3. בסיום תקבלו כתובת קבועה בסגנון https://clinic-rooms-xxxx.vercel.app — זו הכתובת שתמסרו לצוות.
🔁 מרגע זה, כל עדכון עתידי של הקוד שיועלה ל-GitHub ייפרס אוטומטית.

12גיבוי יומי אוטומטי רשות, מומלץ!

שמירת גיבוי מלא של כל הנתונים, כל לילה, למאגר GitHub פרטי נפרד.

  1. האשף יוצר (או מבקש שתיצרו) מאגר פרטי בשם clinic-rooms-backups.
  2. צרו מפתח גישה: גלשו אל יצירת Fine-grained token
  3. מלאו: Token name: clinic-backup · Repository access: Only select repositories → בחרו את מאגר הגיבויים.
  4. Permissions → Repository permissions → Contents → בחרו Read and write.
  5. לחצו Generate token, העתיקו (מוצג פעם אחת בלבד!) והדביקו בחלון השחור.

🎉 אחרי ההתקנה

  • הקובץ SETUP_SUMMARY.txt בתיקיית ההתקנה מרכז את הכתובת ושם המשתמש — שמרו אותו.
  • היכנסו לכתובת המערכת, התחברו, והשלימו את אשף ההגדרה בעברית: שם המרפאה, ימי ושעות פעילות, חדרים ואנשי צוות.
  • מסרו לכל איש צוות את שם המשתמש והסיסמה הזמנית שלו.
  • מדריכים מלאים: docs/ADMIN_GUIDE.md (לניהול) · docs/STAFF_GUIDE.md (לצוות).

הוספת פיצ'רים בעתיד

בדשבורד של Vercel: Project → Settings → Environment Variables → Add, ואחרי כל שינוי: Deployments → ⋯ → Redeploy.

  • עוזר חכם: משתנה ANTHROPIC_API_KEY
  • גיבוי יומי: GITHUB_BACKUP_TOKEN + GITHUB_BACKUP_REPO (בפורמט user/repo)
  • התראות: NEXT_PUBLIC_VAPID_PUBLIC_KEY, VAPID_PRIVATE_KEY, VAPID_SUBJECT (יצירה: npx web-push generate-vapid-keys)

🛠 פתרון תקלות

  • לחיצה על install.bat לא עושה כלום — ודאו שחילצתם את ה-ZIP (לא מריצים מתוכו). אם מופיע מסך כחול "Windows protected your PC" — לחצו More info → Run anyway.
  • "Database connection failed" — העתיקו שוב את כתובת ה-Pooled connection מ-Neon, את כל השורה.
  • הפריסה ל-Vercel נכשלת — ודאו שנרשמתם ל-Vercel עם Continue with GitHub, והריצו את האשף שוב.
  • שכחתם סיסמת ניהול — מנהל אחר מאפס ממסך ניהול המשתמשים.
  • עצרתם באמצע? — פשוט הריצו שוב את install.bat: האשף ממשיך מאיפה שעצר.
